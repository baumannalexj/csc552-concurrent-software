003/**
004 * An iterator over a collection.
005 * This is an implementation of the Iterator pattern from the 
006 * `Gang of Four' book.  It is a simplified version of the
007 * Java <code>Iterator</code> interface, and does not support
008 * deletion.  Collections can return Iterator objects to iterate
009 * over them, for example:
010 * <pre>
011 *   List l = ...;
012 *   for (Iterator i = l.iterator (); i.hasNext ();) {
013 *     Object next = i.next ();
014 *     ...
015 *   }
016 * </pre>
017 * If the underlying list is modified while the iterator is being used,
018 * one of two things may happen:
019 * <ul>
020 * <li>A <code>IteratorModificationException</code> may be thrown.</li>
021 * <li>The iterator will iterate over the elements in the original
022 *   list, not over the elements in the new list.</li>
023 * </ul>
024 * See Lea for a discussion of concurrent updates to collections.
025 * @author Alan Jeffrey
026 * @version 1.0.1
027 * @see List
028 */
029public interface Iterator {
030
031    /**
032     * Are there more elements in the iterator?
033     * @return true if there are more elements in the iterator
034     */
035    public boolean hasNext ();
036
037    /**
038     * The next element in the iterator
039     * @return the next element in the iterator.
040     */
041    public Object next ();
042
043}
